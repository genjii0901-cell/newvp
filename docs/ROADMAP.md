## フェーズ1: ビルドエラーを除去する

**なぜ**: `next.config.ts`で`ignoreBuildErrors: true`になっているため、型エラーが隠れている可能性がある。まず見える状態にする。

**確認方法**:
```bash
npx tsc --noEmit
```

**成功の目印**: エラーゼロで完了する、またはエラー内容が把握できる

**ステータス**: 未着手

## フェーズ2: PDF生成機能を保護・確認する

**なぜ**: PDF生成はコア機能。他の作業で壊さないよう、動作を確認してロック。

**確認方法**:
1. `npm run dev` でローカル起動
2. テストデータで「PDFを作成」ボタンを押す
3. 印刷プレビューが出て、A4 2列レイアウトになっているか確認

**ステータス**: 未着手

## フェーズ3: Supabase認証を確認する

**なぜ**: ログイン・会員管理・使用量記録の基盤。テーブルが存在しないと全て止まる。

**確認手順**:
1. `/check` ページで環境変数の設定状況を確認
2. Supabaseダッシュボードで以下のテーブルが存在するか確認:
   - `profiles`
   - `subscriptions`
   - `pdf_generations`
   - `wordbooks`（任意）
   - `words`（任意）
3. テーブルが存在しない場合は作成（SQL定義は `docs/AI_HANDOFF.md` を参照）
4. ローカルまたはnewvp.vercel.appでログイン・新規登録を試す

**成功の目印**: ログイン後にプラン情報が表示される

**ステータス**: 未着手

---

## フェーズ4: Stripe課金を接続する

**なぜ**: 有料プランへのアップグレードが動かないと収益化できない。

**確認手順**:
1. Stripeダッシュボードで Personal・Teacher の Price IDを確認
2. Vercelの環境変数に `STRIPE_PRICE_PERSONAL` / `STRIPE_PRICE_TEACHER` を設定
3. Webhookエンドポイントを Stripeに登録: `https://newvp.vercel.app/api/stripe/webhook`
4. `STRIPE_WEBHOOK_SECRET` をVercelに設定
5. Stripeのテストモードで決済を試す

**成功の目印**: テスト決済後にプランが「Personal」に更新される

**ステータス**: 未着手
